P7650 题解

题目大意

给你一个排列 pip_ipi​，每次可以将第 iii 个元素移到第 jjj 个位置，花费 i+ji+ji+j 的代价，问最少花费多少代价将这个排列变成 1∼n1\sim n1∼n 的排列。

大体思路

对于一个数字 iii，设它现在的下标为 pip_ipi​。它要移动到正确的位置（即数字 i+1i+1i+1 的前面一个位置），有两种方式：用一次操作将它「主动」移过去，或者把 [pi+1∼pi+1−1][p_i+1\sim p_{i+1}-1][pi​+1∼pi+1​−1] 之间的数字全部移走，让这个数「被动」跑到这个位置。如果 pi<pi+1p_i<p_{i+1}pi​<pi+1​，则用两种方式移动均可。否则，只能使用第一种方式。

我们可以用上面的方法，从 n−1n-1n−1 至 111 移动每个数。由于是「倒序」移动，所以当移动数 iii 时，i+1∼ni+1\sim ni+1∼n 就已经排好序了，且 1∼i1\sim i1∼i 的相对位置和未排序前一样。

我们设 dp(i,j)dp(i,j)dp(i,j) 表示现在要排 iii 这个数，已经排好 i+1∼ni+1\sim ni+1∼n，且 iii 要放到一开始的序列的第 jjj 号位置的最小代价。记 now(x)now(x)now(x) 为数字 xxx 现在的位置，pos(x)pos(x)pos(x) 为数字 xxx 一开始的位置。先来考虑第一种移动方式，我们可以枚举 jjj，即

dp(i,j)=min⁡{dp(i+1,j)+now(i)+now(j)}.dp(i,j)=\min\{dp(i+1,j)+now(i)+now(j)\}.dp(i,j)=min{dp(i+1,j)+now(i)+now(j)}.

注意这里是 now(j)now(j)now(j) 而不是 jjj，因为 jjj 是初始序列的下标。

如果使用第二种移动方式，则需要麻烦一些了。根据徐源盛大佬的论文《对一类动态规划问题的研究》，我们可以知道一个性质：当前决策对未来行动的费用影响只与当前决策有关。也就是说，如果 iii 这个数不动，那么可能对以后的决策花费产生影响，具体一点就是初始位置在 [pos(i)+1,j−1][pos(i)+1, j-1][pos(i)+1,j−1] 之间的数在排序过程中都需要跨越一次 iii。根据论文，我们知道对于整数 x,x<ix,x<ix,x<i，xxx 跨越 iii 对答案所增加的代价是 i−xi-xi−x。由于「当前决策对未来行动的费用影响只与当前决策有关」，我们可以直接把这个 i−xi-xi−x 加进当前决策的花费中。也就是说，对于移动方式二，我们有以下转移方程：

dp(i,pos(i))=min⁡j=pos(i+1)n{dp(i+1,j)+(∑k=pos(i)+1j−1max⁡{i−sk,0})}dp(i,pos(i))=\min_{j=pos(i+1)}^n\{dp(i+1,j)+(\sum_{k=pos(i)+1}^{j-1}\max\{i-s_k,0\})\}dp(i,pos(i))=j=pos(i+1)minn​{dp(i+1,j)+(k=pos(i)+1∑j−1​max{i−sk​,0})}

转移方程里的那个求和可以在转移的过程中记录，所以该 DP 的时间复杂度为 O(n2)O(n^2)O(n2)。

核心代码

memset(dp, 0x3F, sizeof dp);
dp[n][n] = 0;
per(i, n - 1, 1) {
  int posi = 1, posj = 1;
  re(j, pos[i] - 1) posi += (s[j] < i);
  // 移动方式 1
  re(j, n) {
    posj += (s[j] < i);
    if (dp[i + 1][j] != 0x3F3F3F3F) dp[i][j] = dp[i + 1][j] + posi + posj;
  }
  // 移动方式 2
  int sum = 0;
  rep(j, pos[i] + 1, n) {
    if (dp[i][pos[i]] > dp[i + 1][j] + sum) dp[i][pos[i]] = dp[i + 1][j] + sum, pre[i] = j;
    sum += max(0, i - s[j]);
  }
}
